## What We're Building

gbrain 目前只处理纯文本 markdown。用户有一个 ChatGPT 导出：490 个对话，50 个含图片，538 张图像，6 个 PDF。这些内容无法导入。v0.5 的目标：

1. `gbrain attach <page-slug> <file>` — 把文件关联到知识页面
2. `gbrain describe <file-slug>` — 用 vision LLM 生成图片描述（可被 FTS5 + vector 搜索）
3. `gbrain import-chatgpt <export-dir>` — 一次性导入 490 个对话 + 538 张图片

```sql
-- ── Files ────────────────────────────────────────────────────────────────────

CREATE TABLE IF NOT EXISTS files (
  slug          TEXT    PRIMARY KEY,
  sha256        TEXT    NOT NULL,
  file_path     TEXT    NOT NULL,
  original_name TEXT,
  mime_type     TEXT    NOT NULL,
  size_bytes    INTEGER NOT NULL,
  description   TEXT,
  created_at    TEXT    NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))
);

CREATE UNIQUE INDEX IF NOT EXISTS idx_files_sha256 ON files(sha256);

CREATE TABLE IF NOT EXISTS page_files (
  page_slug     TEXT    NOT NULL REFERENCES pages(slug) ON DELETE CASCADE,
  file_slug     TEXT    NOT NULL REFERENCES files(slug) ON DELETE CASCADE,
  display_order INTEGER NOT NULL DEFAULT 0,
  created_at    TEXT    NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now')),
  PRIMARY KEY (page_slug, file_slug)
);

CREATE INDEX IF NOT EXISTS idx_page_files_page ON page_files(page_slug);
CREATE INDEX IF NOT EXISTS idx_page_files_file ON page_files(file_slug);

-- FTS5 index for file descriptions
CREATE VIRTUAL TABLE IF NOT EXISTS fts_files USING fts5(
  description,
  content='files',
  content_rowid='rowid',
  tokenize='porter unicode61'
);

-- Only index when description is not null
CREATE TRIGGER IF NOT EXISTS files_ai AFTER INSERT ON files
WHEN new.description IS NOT NULL
BEGIN
  INSERT INTO fts_files(rowid, description) VALUES (new.rowid, new.description);
END;

CREATE TRIGGER IF NOT EXISTS files_au AFTER UPDATE OF description ON files
WHEN new.description IS NOT NULL
BEGIN
  INSERT INTO fts_files(fts_files, rowid, description)
    VALUES('delete', old.rowid, COALESCE(old.description, ''));
  INSERT INTO fts_files(rowid, description) VALUES (new.rowid, new.description);
END;

CREATE TRIGGER IF NOT EXISTS files_ad AFTER DELETE ON files BEGIN
  INSERT INTO fts_files(fts_files, rowid, description)
    VALUES('delete', old.rowid, COALESCE(old.description, ''));
END;

-- Separate table for file description embeddings (content_chunks.page_id is NOT NULL,
-- cannot be reused for files without a table recreate migration — too risky)
CREATE TABLE IF NOT EXISTS file_chunks (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  file_slug   TEXT    NOT NULL REFERENCES files(slug) ON DELETE CASCADE,
  chunk_index INTEGER NOT NULL DEFAULT 0,
  chunk_text  TEXT    NOT NULL,
  embedding   BLOB,
  model       TEXT    NOT NULL DEFAULT 'text-embedding-3-small',
  embedded_at TEXT,
  created_at  TEXT    NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now')),
  UNIQUE(file_slug, chunk_index)
);

CREATE INDEX IF NOT EXISTS idx_file_chunks_file ON file_chunks(file_slug);
```

## Step 3: New Core Modules

### src/core/files.ts (工具函数，必须 export 供测试使用)

```typescript
import { createHash } from "crypto";
import { extname, basename } from "path";
import { existsSync, readdirSync } from "fs";
import { join } from "path";

// 导出供测试使用
export const MIME_TYPES: Record<string, string> = {
  '.jpg': 'image/jpeg', '.jpeg': 'image/jpeg', '.png': 'image/png',
  '.gif': 'image/gif', '.webp': 'image/webp', '.svg': 'image/svg+xml',
  '.pdf': 'application/pdf', '.mp4': 'video/mp4', '.m4a': 'audio/mp4',
  '.mp3': 'audio/mpeg', '.wav': 'audio/wav', '.heic': 'image/heic',
  '.tiff': 'image/tiff', '.tif': 'image/tiff',
  '.doc': 'application/msword',
  '.docx': 'application/vnd.openxmlformats-officedocument.wordprocessingml.document',
};

export function getMimeType(filePath: string): string | null {
  return MIME_TYPES[extname(filePath).toLowerCase()] ?? null;
}

export function fileHash(content: Buffer): string {
  return createHash('sha256').update(content).digest('hex');
}

// description → kebab slug (max 80 chars), fallback to original_name
export function toFileSlug(text: string): string {
  return text
    .toLowerCase()
    .replace(/[^a-z0-9]+/g, '-')
    .replace(/^-+|-+$/g, '')
    .slice(0, 80);
}

// Find non-colliding filename: react-diagram.png → react-diagram-2.png → ...
export function resolveFileSlug(
  desiredSlug: string,
  ext: string,
  filesDir: string
): string {
  let candidate = desiredSlug;
  let attempt = 1;
  while (existsSync(join(filesDir, `${candidate}${ext}`))) {
    attempt++;
    candidate = `${desiredSlug}-${attempt}`;
    if (attempt > 999) throw new Error(`Too many slug collisions for: ${desiredSlug}`);
  }
  return candidate;
}
```

### src/core/vision.ts (image description via LLM)

**Pattern: 完全复制 src/commands/compile/llm.ts 的 FetchFn 注入模式**

```typescript
import type { Config } from "./config.ts";

// Injectable for tests — same pattern as compile/llm.ts
export type FetchFn = typeof fetch;

export interface VisionConfig {
  base_url?: string;
  api_key?: string;
  model: string;
}

/**
 * Generate a text description of an image using a vision-capable LLM.
 * Throws on timeout, network error, or non-200 response.
 * base64Data: Buffer.from(fileContent).toString('base64')
 */
export async function describeImage(
  base64Data: string,
  mimeType: string,
  cfg: VisionConfig,
  fetchFn: FetchFn = fetch
): Promise<string> {
  const baseUrl = cfg.base_url ?? "https://api.openai.com/v1";
  const model = cfg.model;
  const apiKey = cfg.api_key;

  const controller = new AbortController();
  const timer = setTimeout(() => controller.abort(), 60_000); // 60s for images

  try {
    const res = await fetchFn(`${baseUrl}/chat/completions`, {
      method: "POST",
      headers: {
        "Authorization": `Bearer ${apiKey}`,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        model,
        messages: [{
          role: "user",
          content: [
            {
              type: "text",
              text: "Describe this image in 2-3 sentences. Focus on the main subject, key visual elements, and any text visible. Be specific and informative — this description will be used as searchable text.",
            },
            {
              type: "image_url",
              image_url: { url: `data:${mimeType};base64,${base64Data}` },
            },
          ],
        }],
        max_tokens: 300,
      }),
      signal: controller.signal,
    });

    clearTimeout(timer);

    if (!res.ok) {
      const body = await res.text().catch(() => "");
      throw new Error(`Vision API error ${res.status}: ${body.slice(0, 200)}`);
    }

    const json = await res.json() as { choices?: Array<{ message?: { content?: string } }> };
    const content = json.choices?.[0]?.message?.content;
    if (!content) throw new Error("Vision API returned empty content");
    return content.trim();
  } catch (err) {
    clearTimeout(timer);
    if (err instanceof Error && err.name === "AbortError") {
      throw new Error("Vision API timeout (60s exceeded)");
    }
    throw err;
  }
}
```

### Methods to add to SqliteEngine

```typescript
// ── Files ──────────────────────────────────────────────────────────────────

attachFile(
  pageSlug: string,
  fileRecord: Omit<FileRecord, 'created_at'>
): FileAttachResult {
  // 1. Check page exists
  const page = this.getPage(pageSlug);
  if (!page) throw new Error(`Page not found: ${pageSlug}`);

  // 2. Dedup by sha256
  const existing = this.db
    .query<{ slug: string }, [string]>("SELECT slug FROM files WHERE sha256 = ? LIMIT 1")
    .get(fileRecord.sha256);

  let fileSlug: string;
  let isDuplicate = false;

  if (existing) {
    fileSlug = existing.slug;
    isDuplicate = true;
  } else {
    // 3. Insert new file
    this.db.run(
      `INSERT INTO files (slug, sha256, file_path, original_name, mime_type, size_bytes, description)
       VALUES (?,?,?,?,?,?,?)`,
      [fileRecord.slug, fileRecord.sha256, fileRecord.file_path,
       fileRecord.original_name, fileRecord.mime_type, fileRecord.size_bytes,
       fileRecord.description ?? null]
    );
    fileSlug = fileRecord.slug;
  }

  // 4. Add page_files link (ignore if already linked)
  const maxOrder = this.db.query<{ max: number | null }, [string]>(
    "SELECT MAX(display_order) as max FROM page_files WHERE page_slug = ?"
  ).get(pageSlug)?.max ?? -1;

  this.db.run(
    `INSERT OR IGNORE INTO page_files (page_slug, file_slug, display_order)
     VALUES (?,?,?)`,
    [pageSlug, fileSlug, maxOrder + 1]
  );

  return { slug: fileSlug, isDuplicate };
}

detachFile(pageSlug: string, fileSlug: string, purge = false): void {
  this.db.run(
    "DELETE FROM page_files WHERE page_slug = ? AND file_slug = ?",
    [pageSlug, fileSlug]
  );

  if (purge) {
    const refCount = this.db.query<{ n: number }, [string]>(
      "SELECT COUNT(*) as n FROM page_files WHERE file_slug = ?"
    ).get(fileSlug)?.n ?? 0;

    if (refCount === 0) {
      // Get file path before deleting
      const f = this.db.query<{ file_path: string }, [string]>(
        "SELECT file_path FROM files WHERE slug = ?"
      ).get(fileSlug);

      this.db.run("DELETE FROM files WHERE slug = ?", [fileSlug]);
      this.db.run("DELETE FROM file_chunks WHERE file_slug = ?", [fileSlug]);

      // Delete physical file (caller's responsibility to handle path)
      if (f) return; // return file_path so caller can delete disk file
      // NOTE: Actual disk deletion handled in the command layer, not engine
    }
  }
}

// Returns {file_path} if purge deleted from DB, so command layer can rm disk file
detachFileWithPurge(pageSlug: string, fileSlug: string): string | null {
  this.db.run(
    "DELETE FROM page_files WHERE page_slug = ? AND file_slug = ?",
    [pageSlug, fileSlug]
  );

  const refCount = this.db.query<{ n: number }, [string]>(
    "SELECT COUNT(*) as n FROM page_files WHERE file_slug = ?"
  ).get(fileSlug)?.n ?? 0;

  if (refCount === 0) {
    const f = this.db.query<{ file_path: string }, [string]>(
      "SELECT file_path FROM files WHERE slug = ?"
    ).get(fileSlug);
    this.db.run("DELETE FROM files WHERE slug = ?", [fileSlug]);
    this.db.run("DELETE FROM file_chunks WHERE file_slug = ?", [fileSlug]);
    return f?.file_path ?? null;
  }
  return null;
}

getFile(slug: string): FileRecord | null {
  return this.db.query<FileRecord, [string]>(
    "SELECT * FROM files WHERE slug = ? LIMIT 1"
  ).get(slug) ?? null;
}

listFiles(pageSlug?: string): FileRecord[] {
  if (pageSlug) {
    return this.db.query<FileRecord, [string]>(
      `SELECT f.* FROM files f
       JOIN page_files pf ON pf.file_slug = f.slug
       WHERE pf.page_slug = ?
       ORDER BY pf.display_order`
    ).all(pageSlug);
  }
  return this.db.query<FileRecord, []>(
    "SELECT * FROM files ORDER BY created_at DESC"
  ).all();
}

setFileDescription(fileSlug: string, description: string): void {
  // Verify file exists before update (FTS trigger safety)
  const exists = this.db.query<{ rowid: number }, [string]>(
    "SELECT rowid FROM files WHERE slug = ? LIMIT 1"
  ).get(fileSlug);
  if (!exists) throw new Error(`File not found: ${fileSlug}`);

  this.db.run(
    "UPDATE files SET description = ? WHERE slug = ?",
    [description, fileSlug]
  );
}

upsertFileChunk(fileSlug: string, chunkText: string, embedding: Float32Array, model: string): void {
  const embBlob = Buffer.from(embedding.buffer);
  this.db.run(
    `INSERT OR REPLACE INTO file_chunks
       (file_slug, chunk_index, chunk_text, embedding, model, embedded_at)
     VALUES (?,0,?,?,?,strftime('%Y-%m-%dT%H:%M:%SZ','now'))`,
    [fileSlug, chunkText, embBlob, model]
  );
}

listUnescribedFiles(): FileRecord[] {
  return this.db.query<FileRecord, []>(
    "SELECT * FROM files WHERE description IS NULL ORDER BY created_at"
  ).all();
}
```

### searchKeyword() — extend to include fts_files results

After the existing FTS page search, also query fts_files and merge:

```typescript
searchKeyword(query: string, opts?: SearchOpts): SearchResult[] {
  const limit = opts?.limit ?? 10;
  const results: SearchResult[] = [];

  // Existing: page FTS
  try {
    const ftsRows = ftsSearch(this.db, query, { type: opts?.type, limit });
    for (const r of ftsRows) {
      results.push({
        slug: r.slug, page_id: r.id, title: r.title, type: r.type,
        chunk_text: r.snippet ?? '', chunk_source: 'compiled_truth' as const,
        score: r.score, stale: false, snippet: r.snippet,
      });
    }
  } catch { /* FTS error, continue */ }

  // New: file description FTS
  try {
    type FtsFileRow = { rowid: number; description: string; rank: number };
    const fileRows = this.db.query<FtsFileRow, [string]>(
      `SELECT f.rowid, f.description, fts.rank
       FROM fts_files fts
       JOIN files f ON f.rowid = fts.rowid
       WHERE fts_files MATCH ?
       ORDER BY rank LIMIT ?`
    ).all(query, limit);
    // For each matched file, find associated pages
    for (const fr of fileRows) {
      const file = this.db.query<FileRecord, [number]>(
        "SELECT * FROM files WHERE rowid = ? LIMIT 1"
      ).get(fr.rowid);
      if (!file) continue;
      // Get first page this file is attached to for context
      const pf = this.db.query<{ page_slug: string }, [string]>(
        "SELECT page_slug FROM page_files WHERE file_slug = ? LIMIT 1"
      ).get(file.slug);
      results.push({
        slug: pf?.page_slug ?? `file:${file.slug}`,
        page_id: -1,
        title: file.original_name ?? file.slug,
        type: 'file',
        chunk_text: file.description ?? '',
        chunk_source: 'file_description' as const,
        score: Math.abs(fr.rank),
        stale: false,
        snippet: file.description?.slice(0, 200),
      });
    }
  } catch { /* FTS files error, continue */ }

  return results.slice(0, limit);
}
```

## Step 5: New Commands

### gbrain attach (src/commands/attach.ts)

```
args:
  page-slug: positional, required
  file: positional, required  
  --describe: boolean, default false
  --db: string

flow:
  1. resolveDbPath + openDb
  2. loadConfig → check vision config if --describe
  3. Verify page exists (engine.getPage)
  4. Verify file exists (statSync)
  5. Read file content → fileHash() → getMimeType()
  6. If getMimeType() returns null → error "Unsupported file type"
  7. Resolve semantic slug:
     - if --describe: call describeImage() → toFileSlug(description)
     - else: toFileSlug(basename(filePath, ext))
  8. resolveFileSlug() → find non-colliding filename in ~/.gbrain/files/
  9. Copy file to ~/.gbrain/files/{slug}{ext}
  10. engine.attachFile(pageSlug, { slug, sha256, file_path: relative path, ... })
  11. If --describe AND description: engine.setFileDescription(slug, description)
      Then embed the description → engine.upsertFileChunk()
  12. Print: ✓ Attached {slug}.{ext} to {page-slug}
  
error handling:
  - Vision API failure: print warning, continue with description=null
    "⚠ Vision API failed: {err.message}. Attached without description. Run: gbrain describe {slug}"
  - SHA256 dedup: if isDuplicate → "· File already exists as {existing-slug}, linked to page"
```

### gbrain describe (src/commands/describe.ts)

```
args:
  file-slug: positional, optional (required unless --all)
  --all: boolean
  --force: boolean (re-describe already-described files)
  --db: string

flow for single:
  1. engine.getFile(slug) → if null, error
  2. If description exists AND NOT --force → print "· Already described. Use --force to override."
  3. Read file content → describeImage(base64, mimeType, cfg.vision)
  4. engine.setFileDescription(slug, description)
  5. Embed description → engine.upsertFileChunk()
  6. Print: ✓ Described {slug}: "{description.slice(0, 60)}..."

flow for --all:
  1. engine.listUndescribedFiles() (or all if --force)
  2. Batch N=5 concurrent with 200ms pause between batches
  3. Progress: "Describing X/Y files..."
  4. Each file: same flow as single
  5. On SIGINT: print "Interrupted after X/Y. Resume with: gbrain describe --all"
  6. Final: "✓ Described X files, Y failed"
```

### gbrain import-chatgpt (src/commands/import-chatgpt.ts)

```
args:
  export-dir: positional, required
  --dry-run: boolean
  --describe: boolean (trigger gbrain describe on imported images)
  --db: string

flow:
  1. Verify export-dir exists
  2. Build file index: scan export-dir → Map<fileId, string> (filePath)
     Pattern A: "file-{fileId}-{name}.{ext}" → extract fileId
     Pattern B: "{hash}#file_{numericId}#{page}.{hash}.{ext}" → index by numericId
  3. Find conversations-*.json files (glob export-dir/conversations-*.json)
  4. ENTIRE IMPORT IS ONE TRANSACTION (db.transaction(() => { ... })())
  5. For each JSON file:
     a. JSON.parse — wrap in try/catch per file, skip + warn on parse error
     b. For each conversation:
        - slug = "chatgpt-{conversation_id.slice(0,8)}"
        - Extract title, messages (tree traversal via mapping)
        - Build content: chronological messages as markdown
        - Extract image refs: parts where content_type = "image_asset_pointer"
          → asset_pointer = "file-service://file-{fileId}" → extract fileId
        - engine.putPage(slug, { type: 'session', title, compiled_truth: content })
        - For each image ref: fileIndex.get(fileId) → if found, attach
  6. Print: N pages created/updated, M files attached, K unmatched images
     (even in dry-run, just print what WOULD happen)

ChatGPT message tree traversal:
  The conversations have a `mapping` field (Record<nodeId, {message, parent, children}>).
  To get chronological messages:
  1. Find root node (parent = null OR parent = "client-created-root")
  2. Walk children depth-first, collect message.content items
  3. message.author.role = "user" | "assistant" | "tool"
  4. Include both user and assistant messages

File index building:
  const fileIndex = new Map<string, string>();
  for (const name of readdirSync(exportDir)) {
    // Pattern A: file-{fileId}-anything.ext
    const matchA = name.match(/^file-([A-Za-z0-9]+)-/);
    if (matchA) fileIndex.set(matchA[1], join(exportDir, name));
    // Pattern B: hash#file_{numericId}#page.hash.ext
    const matchB = name.match(/#file_(\d+)#/);
    if (matchB) fileIndex.set(matchB[1], join(exportDir, name));
  }

Critical: handle missing fields gracefully. Some conversations may lack title,
created_at, etc. Always use optional chaining + defaults.
```

## Critical Gaps (MUST fix during implementation)

### Gap 1: import-chatgpt JSON parse errors

**Problem:** If any `conversations-*.json` has malformed JSON or missing fields, the whole import must not crash.

**Fix:** Wrap per-conversation processing in try/catch:
```typescript
let importErrors = 0;
for (const conv of conversations) {
  try {
    // ... process conversation
  } catch (err) {
    importErrors++;
    console.warn(`⚠ Skipped conversation ${conv.id ?? '?'}: ${err.message}`);
  }
}
```

### Gap 2: FTS5 trigger safety in setFileDescription

**Problem:** If `setFileDescription()` is called on a file that doesn't exist (stale reference), the UPDATE triggers will operate on 0 rows silently.

**Fix (already in spec above):** Check `rowid` existence before UPDATE, throw if not found. This is already in the `setFileDescription()` spec.
