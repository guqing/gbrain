# exo v0.7 — Search Quality Fix + 极简 Web UI

> **For Claude Code:** 完整实现 spec，top to bottom 读完再动手。
> 两个独立部分：Part 1（v0.6.3，先做）修复搜索质量；Part 2（v0.7）加 Web UI。

---

## 背景 & 问题

`exo query` 目前有两个独立问题：

**问题1 搜索质量退化**
没有 OpenAI API key 时退化成纯 FTS 关键词搜索。FTS 对图片的 LLM 生成描述权重
虚高，导致图片结果排在语义更相关的 page 内容前面。RRF 分数全部 0.015~0.016，
几乎无差异，无法区分相关性。

**问题2 CLI 显示零散**
snippet 截断到 120 字符太短，关键词往往出现在靠后的位置但 snippet 只显示开头。
`---` 分隔符混入 snippet（compiled_truth 和 timeline 内容混在同一 chunk）。

**用户目标**：个人使用，能快速找到知识并看清上下文，不是 demo 展示。

if (!embedding) {
  for (const [, entry] of scores) {
    if (
      entry.result_kind === "file" &&
      entry.chunk_source === "file_description"

      entry.rrf *= 0.6;

// 然后才排序（这行原本就有，确认顺序正确）

### 1c. Snippet 关键词定位

**文件：`src/commands/query.ts`**

当前代码（约 L157）：
```typescript
const raw = r.chunk_text.replace(/\s+/g, " ").trim();
const snip = raw.length > 120
  ? raw.slice(0, raw.lastIndexOf(" ", 120) || 120) + "…"
  : raw;
```

替换为使用关键词定位的 snippet 函数：

```typescript
function extractSnippet(text: string, query: string, maxLen = 220): string {
  const lower = text.toLowerCase();
  // 逐词尝试，找到第一个命中的词
  const terms = query.toLowerCase().split(/\s+/).filter(Boolean);
  let idx = -1;
  for (const term of terms) {
    const found = lower.indexOf(term);
    if (found !== -1) { idx = found; break; }
  }
  // 任何词都没命中（FTS stemming 命中但原词不在文本）→ 回退到开头
  if (idx === -1) {
    return text.slice(0, maxLen) + (text.length > maxLen ? "…" : "");
  }
  const start = Math.max(0, idx - 60);
  const end = Math.min(text.length, idx + 160);
  return (start > 0 ? "…" : "") + text.slice(start, end) + (end < text.length ? "…" : "");
}

// 使用：
const raw = r.chunk_text.replace(/\s+/g, " ").trim();
const snip = extractSnippet(raw, originalQuery);
```

## Part 2 — 极简 Web UI（v0.7）

### 设计原则

- `exo ui` 是**新命令**，不修改现有 `exo serve`（MCP stdio 保持独立）
- 后端：`Bun.serve()` 原生 HTTP，不引入 Hono 等额外依赖
- 前端：单文件 `src/ui/html.ts` 导出内嵌 HTML 字符串，零构建步骤
- 功能范围：**搜索 + 结果浏览**，本版本不做编辑/compile

### 命令接口

```
exo ui [--db <path>] [--port <number>] [--no-open]
```

默认端口 **7499**（可在 `~/.exo/config.toml` 的 `[ui]` section 配置）。
启动后自动打开浏览器，除非传 `--no-open`。

### 新文件：`src/commands/ui.ts`

```typescript
import { defineCommand } from "citty";
import { openDb, resolveDbPath } from "../core/db.ts";
import { SqliteEngine } from "../core/sqlite-engine.ts";
import { embed } from "../core/embedding.ts";
import { loadConfig } from "../core/config.ts";
import { UI_HTML } from "../ui/html.ts";

export default defineCommand({
  meta: { name: "ui", description: "Start local web UI for knowledge browsing" },
  args: {
    db: { type: "string", description: "Path to brain.db" },
    port: { type: "string", description: "Port (default 7499)" },
    "no-open": { type: "boolean", description: "Don't open browser", default: false },
  },
  async run({ args }) {
    const cfg = loadConfig(args.db ? { db: args.db } : undefined);
    const db = openDb(resolveDbPath(args.db));
    const engine = new SqliteEngine(db);
    const port = parseInt(args.port ?? "7499");

    // 跨平台 open
    const opener =
      process.platform === "darwin" ? "open"
      : process.platform === "win32" ? "start"
      : "xdg-open";

    Bun.serve({
      port,
      async fetch(req) {
        const url = new URL(req.url);

        if (url.pathname === "/" || url.pathname === "/index.html") {
          return new Response(UI_HTML, {
            headers: { "Content-Type": "text/html; charset=utf-8" },
          });
        }

        try {
          // GET /api/search?q=...&type=page|session|inbox
          if (url.pathname === "/api/search") {
            const q = url.searchParams.get("q") ?? "";
            if (!q) return Response.json([]);
            const rawType = url.searchParams.get("type");
            const validTypes = ["page", "session", "inbox", "research"];
            const type = rawType && validTypes.includes(rawType) ? rawType : undefined;
            const limit = Math.min(parseInt(url.searchParams.get("limit") ?? "20"), 100);
            // embed() は内部で config を読む、引数は text のみ
            const embedding = await embed(q).catch(() => null);
            const results = engine.hybridSearch(q, embedding, { limit, type });
            return Response.json(results);
          }

          // GET /api/page/:slug
          if (url.pathname.startsWith("/api/page/")) {
            const slug = decodeURIComponent(url.pathname.slice("/api/page/".length));
            const page = engine.getPage(slug);
            if (!page) return new Response("Not found", { status: 404 });
            return Response.json(page);
          }

          return new Response("Not found", { status: 404 });
        } catch (e) {
          return Response.json({ error: String(e) }, { status: 500 });
        }
      },
    });

    // 优雅关闭：Ctrl-C 时关闭 DB
    process.on("SIGINT", () => { db.close(); process.exit(0); });

    console.log(`exo UI → http://localhost:${port}`);
    if (!args["no-open"]) {
      Bun.spawn([opener, `http://localhost:${port}`]);
    }
    await new Promise(() => {}); // keep alive
  },
});
```

### 新文件：`src/ui/html.ts`

导出一个 `UI_HTML` 字符串常量，包含完整的单页 HTML。结构：

```
┌──────────────────────────────────────────────────────┐
│ 🧠 exo                            [    search...    ] │
├──────────────────────────────────────────────────────┤
│ [All] [Pages] [Sessions] [Files]          10 results  │
├──────────────────────────────────────────────────────┤
│ ● Redis 限流实现方案              page  ████░░ 0.024  │
│   …令牌桶算法在 Redis 中用 SETNX+EXPIRE 实现，        │
│   过期时间设为时间窗口长度…                            │
│                                                      │
│ ● 统计API调用次数                 session  ███░░ 0.022│
│   …将调用记录到 Redis 或数据库，使用 INCR 方法统计…   │
│                                                      │
│ 🖼 SCR-20250208-kypy.png          file   █░░░░ 0.012  │
│   The image displays a user interface…               │
└──────────────────────────────────────────────────────┘
```

**关键设计决策：**

1. **Tab 过滤规则**
   - `[Pages]` → 请求 `/api/search?type=page`（server-side，PageType 过滤）
   - `[Sessions]` → 请求 `/api/search?type=session`
   - `[Files]` → 请求 `/api/search`（不加 type），**在前端** 按 `result_kind === "file"` 过滤
     （file results 不是 page，type 字段不适用，必须客户端过滤）

2. **相对分数条**
   - `barWidth = result.score / results[0].score * 100 + "%"`
   - 用相对宽度，直观看出第一名和后续的差距

3. **Snippet 显示**：不截断，完整显示 chunk_text（已在 Part 1 改过分块逻辑）

4. **图片类结果**：显示 🖼 图标 + 文件名 + 描述文字，不显示缩略图
   （二进制文件 serving 是 future work）

5. **点击展开全文**：点击任意结果调用 `/api/page/:slug`，在下方 inline 展开
   `page.content`（Markdown 渲染用 `<pre>` 暂时 fallback，后续加 marked.js）

6. **实时搜索**：`input` 事件 + `debounce(300ms)`，输入即搜

**前端需要的 SearchResult 字段**（来自 `hybridSearch()` 返回）：

| 字段 | 类型 | 用途 |
|------|------|------|
| `slug` | string | 展开全文的 key |
| `title` | string \| null | 显示名称 |
| `chunk_text` | string | snippet 来源 |
| `score` | number | 分数条计算 |
| `result_kind` | "page" \| "file" | 图标 + Files tab 过滤 |
| `type` | string | Pages/Sessions tab 分类 |
| `chunk_source` | string \| null | 来源标识（可选展示） |

## Open Questions（已明确处理方式）

1. **图片缩略图**：本版本不实现 binary file serving，只显示文字描述。
   如需预览，后续加 `GET /api/file/:slug/raw` 端点。

2. **端口冲突**：Bun.serve() 端口冲突时抛出错误，让用户用 `--port` 指定。
   不做自动递增，行为可预期更重要。

3. **`localhost` 安全**：无认证，仅绑定 localhost（Bun.serve 默认行为）。
   公网服务器上运行时需用 SSH tunnel，文档注明即可。

4. **分数折扣系数**：`0.6` 是初始值，基于实际 DB 验证后调整。
